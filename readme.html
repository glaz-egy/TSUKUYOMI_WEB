管理者、ユーザに限らずサイト左上にある三本線のマークをクリックすることでページを移動できます。
サイドメニューの一番下の項目ではカラーテーマを設定する事が出来ます(Internet Explorerには非対応。Microsoft Edge, Google Chrome, Mozilla Firefox, Vivaldiにて動作確認済み)。
カラーテーマの設定はブラウザを閉じても保持されます。

管理者向け

概要

管理者はユーザから送信された情報を整理した状態で閲覧、保存する事が出来ます。
ソート、受信する項目の選択、受信したデータの保存を一ページ内で行えます。

ソートについて

ソートは時間、社員番号と昇順、降順の組み合わせで行えます。ソート内容を設定したあと、ソートボタンを押すことでソートされます。
ソート内容の設定は「ソート順」と書かれたプルダウンメニューで可能です。 また、ソートした結果は保存するデータにも引き継がれます。

社員番号重複許可と受付種別について

社員番号重複許可の設定項目では社員番号重複を設定します。それぞれを設定した場合の動作は以下の表の通りです。

設定 内容
社員番号の重複を許可しない 同じ社員番号が送信されてきた場合、先に送られてきた方を優先し、以降送られてきたものについては破棄します。
デフォルトの設定値です。
異なる種別の場合に社員番号の重複を許可する 同じ社員番号でも種別が異なっていた場合、それぞれ最初のものは受信します。
種別、社員番号がすでに存在している場合は上記同様、破棄します。
社員番号の重複を許可する 種別、社員番号が重複していた場合でも一切破棄せず、全て受信します。

次に受付種別では受信をする種別について設定できます。
チェックを入れている種別については受信を行い、入れていないものは破棄します。
これらの設定はブラウザを閉じる、または再読込みをおこなっても保持されます。

各種ボタンについて

リセットボタンは表示をリセットします。
ソートボタンはソートを行います。
取得ボタンは受信したデータをファイルに保存します。※IEは右クリックで保存
保存形式は以下の例の通りです。
[2020/04/13 01:01:36]xxxx, yyyy, 出勤
[2020/04/13 01:01:40]zzzz, aaaa, 出勤
[2020/04/13 01:01:45]bbbb, cccc, 退勤
[2020/04/13 01:04:16]bbbb, cccc, 100, テスト

ユーザ向け

概要

ユーザは出勤、退勤、その他のボタンを押すことで管理者に情報を送信できます。
また、クエリパラメータにname, number, typeを含めることによって手間無く利用可能です。

クエリパラメータについて

クエリパラメータのnameには氏名、numberには社員番号をそれぞれ入力します。 typeは下記表の項目を入力します。

項目名 内容
join 出勤の情報をサーバ接続時、または管理者がログインしたら送信します。
正常に送信が行えた場合、ボタンの上部に「GETパラメータにより出勤しました」が表示されます。
leave 退勤の情報をサーバ接続時、または管理者がログインしたら送信します。
正常に送信が行えた場合、ボタンの上部に「GETパラメータにより退勤しました」が表示されます。
other 氏名、社員番号にあらかじめ入力された状態になります。
以上より、出勤を即時に行うためのクエリパラメータを含んだURLは次のようになります。
http://www.tsukuyomi.work/?name=xxxx&number=yyyy&type=join //xxxx, yyyyはそれぞれ氏名, 社員番号

それぞれのボタンについて

ボタンはクエリパラメータを用いない場合の他、その他やテストの点数を送る際に使用します。
出勤、退勤を送信する場合には氏名、社員番号の入力が必須です。
その他ボタンは出勤、退勤以外で一時的に退出、または離席した場合に送信する際に使用します。
点数を入力してその他ボタンを押すことで、テストの点数を送信する事が出来ます。
